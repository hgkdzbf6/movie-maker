# Movie Maker Roadmap & Test Plan

本文件描述当前项目距离“类似 Windows Movie Maker 的视频剪辑软件”还差哪些能力，并给出分阶段的开发与测试计划。

## 当前状态（已实现）

- 构建与测试：`npm run build` 可通过；`npm test` 全绿。
- 开发效率：`npm run dev:all` 一键启动（前端+调度器）；`npm run dev:restart` 清理并重启。
- 时间轴交互：素材拖拽落点、吸附预览、播放头 scrub、播放时自动滚动跟随、片段鼠标拖动。
- 音频体验：音频轨视觉（波形）、防重叠自动避让；添加视频素材时自动创建对应音频片段。
- 预览：Remotion composition 根据 `tracks/assets` 渲染 `Img/Video/Audio`；Player 事件与 store 同步；播放到末尾停止。
- 调试/工程：一键清空（调试）；工程 JSON 保存/加载（version=1）。

## 与“真正 Movie Maker”相比的差距

按用户可感知影响从高到低排序：

1) 编辑基础能力（核心缺口）
- 片段裁剪（trim）、滑移（ripple）、分割/合并/吸附到片段边缘与播放头。
- 多轨：多条视频轨、多条音频轨、轨道独立高度可拖拽。
- 文本/字幕轨、转场、基础特效（裁剪/缩放/位置/旋转/透明度）。

2) 工程与素材体系（可用性缺口）
- 素材持久化：当前工程文件可能包含 `blob:` URL，跨会话不可复用。
- 元数据：时长/分辨率/音频采样率等需要真实提取与缓存。
- 工程版本迁移、缺失素材提示与替换。

3) 预览与导出一致性（质量缺口）
- 预览/导出需要同一套 timeline 数据模型与渲染链路。
- 长时间轴性能（虚拟化、节流、波形缓存策略）。

4) 产品完成度（效率缺口）
- 撤销/重做、快捷键、吸附 UI、时间刻度/缩放体验、导出面板与错误提示。

## 里程碑计划

时间仅为建议（按 1 人开发估算）。每个里程碑都给出验收标准。

### Milestone 0：稳定性收敛（1–2 天）

目标：把“现在已有的”变得稳定、可持续迭代。

范围：
- 工程加载后 `tracks/scenes/assets` 一致性校验与修复（避免漂移/重复）。
- 缺失素材处理：被引用素材缺失时预览不崩溃、时间轴有明显提示。
- 波形解码：并发限制、失败回退、缓存上限与释放策略。

验收：
- 10 分钟连续拖拽/播放/缩放不崩溃、不明显卡顿。
- 缺失素材工程能打开、可导出前给明确提示。

### Milestone 1：可用编辑 MVP（1–2 周）

目标：完成“导入→剪辑→预览一致→导出”的最小闭环。

范围：
- 时间轴编辑：trim（左右手柄裁剪）、分割、删除、复制、移动。
- 吸附：对齐到播放头/片段边缘（可配置阈值）。
- 轨道：支持多条视频轨/音频轨；轨道独立高度可拖拽调整。
- 音频：音量、淡入淡出（in/out）。
- 工程/素材：将 `blob:` 升级为持久化资源（例如 `/uploads` + 工程目录）；工程 JSON 记录相对路径与元数据。
- 导出：基于同一 timeline 模型渲染；导出前检查缺失素材并阻止。

验收：
- 可完成一个短片：导入 mp4 + 配音 + 图片 → 排时间轴 → trim/分割 → 预览一致 → 导出 mp4（含音频）。

### Milestone 2：接近 Movie Maker v1（2–4 周）

目标：提升编辑效率与表达能力。

范围：
- 文本/字幕轨：基础样式（字体/颜色/描边/阴影）、时长与位置。
- 转场：至少 3 种（淡入淡出/推入/缩放），可调时长。
- 撤销/重做：覆盖所有编辑操作（move/trim/split/delete/add）。
- 快捷键：space、delete、cmd/ctrl+z/y、方向键逐帧/逐秒。
- 性能：时间轴可视区域虚拟化渲染；波形预计算与持久化缓存。

验收：
- 5 分钟内制作“带字幕+转场+配乐”的 10–20 秒视频，不需要刷新页面。

### Milestone 3：更像专业工具 v2（持续迭代）

范围（按优先级可选做）：
- 音频包络线（keyframe）、多轨混音、静音/solo、侧链/降噪/均衡（先做预设）。
- 代理文件（proxy）与后台转码。
- 更智能吸附（关键帧/节拍点/标记点）、磁吸强度。
- 素材管理：标签、搜索、替换素材、批量导入。
- 工程版本：变更记录、自动保存、冲突处理（若有协作）。

## 测试计划

### 单元测试（Jest）

目标：数据结构与纯逻辑不回归。

- Timeline 计算：frame/time、吸附、避让/防重叠算法。
- 工程序列化：`exportProjectFile/loadProjectFile` 兼容性、版本迁移、非法输入。
- 波形：缓存命中/失败回退/并发限制逻辑（可把解码与缓存策略抽成纯函数）。

### 集成测试（Jest）

目标：store + 组件关键路径可验证。

- 拖拽 drop：素材 payload → 创建 scene → 自动进入正确轨道。
- 视频 drop：自动创建对应音频片段。
- 删除素材：引用片段进入 missing 状态，预览不崩溃。

### 端到端测试（Playwright，建议新增）

目标：真实用户操作链路稳定。

- 导入素材 → 拖拽到时间轴 → 播放（播放头移动+自动滚动） → trim/split → 导出。
- 轨道高度拖拽、边栏宽度拖拽（持久化）。
- 保存工程 → 刷新 → 加载工程 → 预览一致。

### 性能/稳定性测试

- 生成 200+ 片段/20min 时间轴：滚动、拖拽、播放的 CPU/内存与交互延迟。
- 长时间播放：`currentFrame` 与播放头位置不漂移。

## 风险与建议

- `blob:` 素材跨会话不可用：优先改为持久化存储（Milestone 1）。
- timeline 模型需要统一：避免“预览一套、导出一套”。
- 大项目性能：尽早做虚拟化与节流，避免后期返工。

## 下一步建议（最短路径）

1) 素材持久化（替换 blob URL）+ 工程文件可复现。
2) trim/split/吸附到片段边缘（Movie Maker 最低门槛）。
3) Playwright 先覆盖“导入→拖拽→播放→导出”的主链路。
