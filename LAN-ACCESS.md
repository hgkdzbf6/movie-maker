## 🔧 技术细节

### Next.js 开发服务器参数

**`-H 0.0.0.0`** 参数说明：
- `-H` 或 `--hostname`: 设置主机名
- `0.0.0.0`: 监听所有网络接口（IPv4）
- 允许局域网内的其他设备访问

### 端口说明

**当前端口**: 3001
- Next.js 检测到端口 3000 被占用，自动使用 3001
- 如果需要使用特定端口，可以添加 `-p 端口号` 参数

### 完整启动命令

```bash
# 使用特定端口
npm run dev -- -p 3000

# 使用自定义主机和端口
npm run dev -- -H 0.0.0.0 -p 3000
```

## 🔒 防火墙配置

如果无法访问，可能需要配置防火墙：

### Ubuntu/Linux

```bash
# 允许 3001 端口（如果使用 ufw）
sudo ufw allow 3001

# 查看防火墙状态
sudo ufw status
```

### Windows 防火墙

1. 打开 Windows 安全中心
2. 防火墙和网络安全 → 允许应用通过防火墙
3. 允许 Node.js 或端口 3001

## 🌍 外网访问

如果需要从外网访问，可以使用之前配置的 FRP 内网穿透：

### FRP 配置

在 FRP 客户端配置中添加：

```toml
[[proxies]]
name = "video-editor"
type = "tcp"
localIP = "127.0.0.1"
localPort = 3001
remotePort = 19001  # 外网访问端口
```

### 外网访问地址

```
http://786100557.xyz:19001/editor
```

## ⚠️ 注意事项

1. **安全性**
   - 此配置仅用于开发环境
   - 生产环境不应使用 `0.0.0.0`
   - 生产环境应使用 Nginx 等反向代理

2. **性能**
   - 局域网访问可能比本地访问稍慢
   - 建议在开发时使用有线连接

3. **兼容性**
   - 确保局域网内的设备兼容
   - 某些企业网络可能有防火墙限制
